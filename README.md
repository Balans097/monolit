# Monolit — GTK4-приложение для стабилизации видео (libvidstab)

Убирает дрожание камеры в видео с помощью двухпроходного алгоритма
`vid.stab` (`vidstabdetect` → `vidstabtransform`) поверх FFmpeg.

Подробное описание возможностей, интерфейса и обоснование архитектурных
решений — см. [manual.md](./manual.md).

## Технический стек

- **Язык**: Nim (`--threads:on`)
- **GUI**: GTK4 (прямые FFI-биндинги через GObject/GLib, без обёрточных
  библиотек), системная динамическая зависимость
- **Обработка видео**: FFmpeg (libavformat/libavcodec/libavfilter),
  libx264, libvidstab — собираются из исходников как статические
  библиотеки (`.a`) и линкуются в бинарник напрямую

## Архитектура

- Стабилизация — два последовательных прохода FFmpeg-фильтров
  (`vidstabdetect` → `vidstabtransform`) по всему файлу целиком;
  распараллеливание на уровне разбиения видео на куски невозможно, т.к.
  `vidstabtransform` сглаживает траекторию с учётом соседних кадров.
  Параллелизм используется на уровне декодера/энкодера/фильтра FFmpeg
  (`thread_count`/`thread_type`).
- Обработка выполняется в отдельном потоке ОС, не блокируя событийный
  цикл GTK; прогресс передаётся в интерфейс через таймер опроса.
- После инициализации GTK принудительно переустанавливается
  `LC_NUMERIC=C` (без изменения остальных категорий локали) —
  необходимо для корректного разбора числовых аргументов
  FFmpeg-фильтров в локалях с десятичной запятой.
- Компрессия `libx264`: режим CRF (1 проход) либо целевой битрейт
  (2 прохода).

## Структура проекта

```
Monolit/
├── Monolit.nim         — точка входа: приглушает лог FFmpeg, запускает gui.runApp()
├── config.nims         — автосборка vidstab+x264+FFmpeg, флаги линковки
├── src/
│   ├── gui.nim         — GTK4-интерфейс: 4 вкладки, обработчики сигналов,
│   │                     чтение значений виджетов в конфигурацию обработки
│   ├── gtk4_api.nim    — Nim-биндинги GTK4/GObject/GLib (FFI)
│   ├── ffmpeg_api.nim  — Nim-биндинги FFmpeg (FFI)
│   └── stabilizer.nim  — двухпроходный пайплайн vidstabdetect→vidstabtransform
├── README.md
└── manual.md
```

## Сборка

```bash
# Fedora, нативно
nim c -d:release --threads:on Monolit.nim

# Windows, кросс-компиляция с Fedora (mingw-w64)
nim c -d:release --threads:on --os:windows Monolit.nim
```

Требования:

- Fedora: `gtk4-devel` (сборка), `gtk4` (запуск)
- Кросс-сборка под Windows: `mingw64-gtk4` (рантайм GTK4 для mingw)
- vid.stab, x264, FFmpeg собираются автоматически из исходников при
  первой сборке средствами `config.nims`; кэш — в `vidstab_build*/`,
  `x264_build_windows/`, `ffmpeg_build*/`

Подробности сборки, известные проблемы кросс-компиляции и их причины —
см. [manual.md](./docs/manual.md#сборка-подробности).
